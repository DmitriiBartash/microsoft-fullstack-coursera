# Activity: Pseudocode

**Course 1 — Foundations of Coding Full-Stack** · Module 2 · Lesson 5 · `You Try It!`

> Practise turning a plain-English problem into **pseudocode** — language-agnostic,
> step-by-step logic — before writing any real code. You follow a fixed three-part
> method (**define → identify processes → write pseudocode**) and apply it to two
> short problems built around the **Input → Processing → Output** model.

---

## 🎯 Objective

Learn to translate a problem statement into clear pseudocode by **defining the problem**,
**identifying its key processes**, and **expressing the logic** as simple, ordered steps —
independent of any programming language.

---

## 📦 What you will produce

For each problem, a short worked solution in **three parts**:

| Part                      | What it captures                                                  |
| ------------------------- | ---------------------------------------------------------------- |
| **1. Define the problem** | A one-sentence restatement of what the program must do           |
| **2. Identify processes** | The **Input**, **Processing**, and **Output** the problem needs  |
| **3. Pseudocode**         | Ordered, plain steps between `Start` and `End`                   |

**Model:** `Input  →  Processing  →  Output` — every problem below maps onto these three stages.

---

## 🧭 Walkthrough

Two problems are worked end-to-end below. Notice how each one follows the **same**
define → identify → pseudocode rhythm.

### Problem 1 — Calculate Total Marks

**Problem statement:** Write pseudocode for a program that calculates the total marks for a
student across three subjects — **Math**, **Science**, and **English**. The program takes the
marks for each subject as input, calculates the total, and prints it.

**1. Define the problem**

You need a program that takes the marks for three subjects (Math, Science, English) as input,
calculates the total marks, and prints the result.

**2. Identify key processes**

- **Input:** Get marks for Math, Science, and English.
- **Processing:** Add the three marks to compute the total.
- **Output:** Display the total marks.

**3. Pseudocode**

```text
Start
    Get Math marks
    Get Science marks
    Get English marks
    Set total to Math marks + Science marks + English marks
    Print total
End
```

### Problem 2 — Greet the User by Name

**Problem statement:** Write pseudocode for a program that takes a user's name as input and
greets them with a message that says `"Hello, [name]!"`.

**1. Define the problem**

Write a program that asks for the user's name and prints a greeting message in the format:
`"Hello, [name]!"`.

**2. Identify key processes**

- **Input:** Get the user's name.
- **Processing:** Construct the greeting message.
- **Output:** Display the greeting message.

**3. Pseudocode**

```text
Start
    Get user's name
    Set greeting to "Hello, " + user's name + "!"
    Print greeting
End
```

## 🔑 Key concepts

- **Pseudocode is language-agnostic** — it describes *logic* in plain steps, so the same
  solution can be implemented in any programming language.
- **Define before you design** — restating the problem in one sentence clarifies exactly what
  the program must accomplish before you write a single step.
- **Input → Processing → Output** — almost every small program decomposes into getting data,
  transforming it, and showing a result; spotting these three stages structures the logic.
- **Plain, ordered actions** — verbs like `Get`, `Set`, and `Print`, one per line between
  `Start` and `End`, keep the flow unambiguous and easy to trace.
- **A repeatable method scales** — the same define → identify → write approach works for a
  two-line greeting or a far larger program.
