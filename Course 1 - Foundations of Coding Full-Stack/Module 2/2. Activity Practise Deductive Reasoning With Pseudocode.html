Practise Deductive Reasoning With Pseudocode

Turn plain-language problems into algorithms using deductive reasoning: state the logical rules (premises), analyse how they relate, draw the control flow that follows, test it against sample inputs, then write the pseudocode. You'll walk through two worked problems — classifying a number and checking a senior discount — and then try one of your own.

🎯 Objective

Practise translating a problem statement into correct algorithm logic by reasoning deductively — moving from a set of stated premises to a guaranteed conclusion — and expressing that conclusion as language-agnostic pseudocode built from If / Else if / Else decisions.

🧩 What you'll produce

For each problem, a short written analysis plus a block of pseudocode, following the same five-step reasoning structure:

1. Identify the premisesList the logical rules the problem gives youA set of If … then … rules
2. Analyse the premisesCheck how the rules relate (mutually exclusive? exhaustive?)A chosen control structure
3. Draw a conclusionOrder the checks into a logical flowBranch order
4. Test the conclusionTrace sample inputs through the flowInput → Output pairs
5. Write the pseudocodeExpress the flow as readable stepsFinal pseudocode

Flow

Premises → Analysis → Logical flow → Test cases → Pseudocode

🚶 Walkthrough

1 Integers (positive / negative / zero)

Write pseudocode for a program that determines whether a number is positive, negative, or zero. Use deductive reasoning to establish the logical flow.

1. Identify the premises (logical rules)

  • If a number is greater than 0, it is positive.
  • If a number is less than 0, it is negative.
  • If a number is equal to 0, it is zero.

2. Analyse the premises

These conditions are mutually exclusive — a number can only fall into one of the three categories. They are also exhaustive — every number is covered. That lets us check them with a single If / Else if / Else chain.

3. Draw a conclusion (logical flow)

  • First, check if the number is greater than 0 → print "Positive".
  • Then check if it is less than 0 → print "Negative".
  • If neither, it must be 0 → print "Zero".

4. Test the conclusion

InputOutput
10Positive
-7Negative
0Zero

5. Pseudocode

Start
    Prompt user to enter a number
    Read number
    If number > 0 then
        Display "Positive"
    Else if number < 0 then
        Display "Negative"
    Else
        Display "Zero"
    End If
End

2 Senior discount

Write pseudocode for a program that checks whether a person is eligible for a senior citizen discount. The program takes the person's age as input. If the age is 65 or older, print "Eligible for senior discount"; otherwise, print "Not eligible for senior discount".

1. Identify the premises (logical rules)

  • If a person's age is 65 or older, they are eligible for a senior discount.
  • If the age is less than 65, they are not eligible.

2. Analyse the premises

There are only two mutually exclusive outcomes based on age — 65 and above → eligible; below 65 → not eligible. A single If / Else is enough; no Else if is needed. Note the boundary: 65 itself is eligible because the rule uses >=.

3. Draw a conclusion (logical flow)

  • If age >= 65 → print "Eligible for senior discount".
  • Else → print "Not eligible for senior discount".

4. Test the conclusion

InputOutput
70Eligible
50Not eligible
65Eligible

5. Pseudocode

Start
    Prompt user to enter age
    Read age
    If age >= 65 then
        Display "Eligible for senior discount"
    Else
        Display "Not eligible for senior discount"
    End If
End

✍️ Your turn

Pick a new problem and run the same five steps end to end. A good candidate:

Grade classifier — read a test score from 0 to 100 and print the letter grade: 90+ → "A", 80–89 → "B", 70–79 → "C", 60–69 → "D", below 60 → "F".

As you work it:

  • Write the premises as explicit If … then … rules, one per grade band.
  • Confirm the bands are mutually exclusive and exhaustive, then order your Else if checks from highest to lowest so each branch only handles its own range.
  • Choose boundary test inputs on purpose — e.g. 90, 89, 60, 59 — not just values in the middle of a band.
  • Finish with pseudocode in the same Start … End / If / Else if / Else style.

🟢 What good looks like

Every premise from the problem statement appears as an explicit rule before any code; the chosen control structure matches the analysis (If / Else if / Else for three or more cases, plain If / Else for exactly two); branch order is deliberate and boundary values (like 0 or 65) land in the intended branch; test cases cover each outcome and the boundaries as input → output pairs; and the final pseudocode is language-agnostic and readable, with clear Start / End and a closed End If.

☑️ Definition of done

  • Premises listed as explicit If … then … rules for each problem
  • Premises analysed as mutually exclusive / exhaustive, and a control structure chosen
  • Logical flow drawn with a deliberate branch order
  • Conclusion tested with sample inputs, including boundary values, as input → output pairs
  • Final pseudocode written in Start … End form with If / Else if / Else and End If
  • "Your turn" grade-classifier problem worked through the same five steps

🔑 Key concepts

  • Deductive reasoning — start from stated premises (the rules) and follow them to a conclusion that must be true; this is exactly how branch logic is derived.
  • Mutually exclusive vs. exhaustive — recognising that cases can't overlap and that they cover every input tells you whether you need Else if chains and a final Else.
  • Branch order matters — with If / Else if / Else, only the first matching condition runs, so ordering (and the final catch-all Else) determines correctness.
  • Boundary conditions — the comparison operator decides the edge case: >= 65 includes 65, while > 65 would exclude it; always test the boundary.
  • Pseudocode is language-agnostic — it captures the logic (prompt, read, decide, display) independent of syntax, so it translates cleanly into any programming language.
