Implementing Conditional and Binary Decision Structures

Work through three small problems and, for each one, decide which conditional structure fits: a binary if/else when there are exactly two outcomes, or a chained if / else if / else when you are classifying among several. You follow the same four-step method every time — define the problem, identify the processes, choose the structure, then write the pseudocode — and finish by turning each design into runnable C#.

🎯 Objective

Learn to read a problem statement, judge whether the decision is binary (two outcomes) or a multi-way classification (several outcomes), and pick the matching control-flow structure — then express that decision first as pseudocode and then as working C# code.

🧩 What you will produce

Three designed-and-implemented solutions, each derived with the same method:

🛠️ Walkthrough

1 Problem 1 — Discount eligibility

Determine whether a customer qualifies for a discount based on how much they have spent. Spend $100 or more → print 10% discount applied.; otherwise → print No discount.

Define the problem

Decide whether a customer qualifies for a discount based on their total spending.

Identify key processes

  • Input the total spending amount — the value the decision is based on.
  • Check the spending condition — is it >= 100?
  • Output the result — apply a discount or not.

Decide on a structure

  • Multiple conditions? No — only one: >= $100.
  • Is the decision binary? Yes — either "discount" or "no discount".
  • Appropriate structure? A simple if / else binary decision.

Pseudocode

Create variable totalAmount and set it to the value of the user input for "Enter total spending amount:"
IF totalAmount is greater than or equal to 100 THEN
    Display to user "10% discount applied."
Otherwise
    Display to user "No discount."

C# implementation

Console.Write("Enter total spending amount: ");
double totalAmount = double.Parse(Console.ReadLine());

if (totalAmount >= 100)
{
    Console.WriteLine("10% discount applied.");
}
else
{
    Console.WriteLine("No discount.");
}

2 Problem 2 — Book categorization

A library categorizes books by genre. Map Fiction, Non-Fiction, and Science Fiction to their categories; anything else → Category: Unknown.

Define the problem

Classify books into categories based on the genre the user provides.

Identify key processes

  • Input the genre — get the genre name from the user.
  • Check the genre against known categories — compare the input to predefined options.
  • Output the category — show the matching classification.

Decide on a structure

  • Multiple conditions? Yes — several genres.
  • Is the decision binary? No — more than two outcomes.
  • Does it involve classification? Yes.
  • Appropriate structure? A sequence of if / else if / else statements.

Pseudocode

Create variable genre and set it to the value of the user input for "Enter the genre of the book:"
IF genre is equal to "Fiction" THEN
    Display to user "Category: Fiction."
Otherwise IF genre is equal to "Non-Fiction" THEN
    Display to user "Category: Non-Fiction."
Otherwise IF genre is equal to "Science Fiction" THEN
    Display to user "Category: Science Fiction."
Otherwise
    Display to user "Category: Unknown."

C# implementation

Console.Write("Enter the genre of the book: ");
string genre = Console.ReadLine();

if (genre == "Fiction")
{
    Console.WriteLine("Category: Fiction.");
}
else if (genre == "Non-Fiction")
{
    Console.WriteLine("Category: Non-Fiction.");
}
else if (genre == "Science Fiction")
{
    Console.WriteLine("Category: Science Fiction.");
}
else
{
    Console.WriteLine("Category: Unknown.");
}

3 Problem 3 — Even or odd number

Determine whether a given number is even or odd. Even → print Even number.; odd → print Odd number.

Define the problem

Determine if the user-provided number is even or odd.

Identify key processes

  • Input the number — get a whole number from the user.
  • Check even or odd — use the modulo operator to test divisibility by 2.
  • Output the result — print whether it is even or odd.

Decide on a structure

  • Multiple conditions? No — only one: divisible by 2 or not.
  • Is the decision binary? Yes.
  • Appropriate structure? A binary if / else decision.

Pseudocode

Create variable number and set it to the value of the user input for "Enter a number:"
IF number modulo 2 is equal to 0 THEN
    Display to user "Even number."
Otherwise
    Display to user "Odd number."

C# implementation

Console.Write("Enter a number: ");
int number = int.Parse(Console.ReadLine());

if (number % 2 == 0)
{
    Console.WriteLine("Even number.");
}
else
{
    Console.WriteLine("Odd number.");
}

▶️ What good looks like

The structure matches the decision: if / else whenever there are exactly two outcomes (Problems 1 and 3), and chained if / else if / else for multi-way classification (Problem 2). Each program reads input, evaluates one clear condition path, and prints exactly one result — the output strings match the spec verbatim (e.g. 10% discount applied., Category: Unknown.), and you can name why a structure was chosen, not just that it works.

☑️ Definition of done

  • Problem 1 designed as if / else and prints 10% discount applied. for >= 100, else No discount.
  • Problem 2 designed as if / else if / else mapping the three genres, with Category: Unknown. as the fallback
  • Problem 3 designed as if / else using number % 2 == 0 to print Even number. / Odd number.
  • Each solution has both pseudocode and a runnable C# implementation
  • For every problem you can state whether the decision is binary or a classification and justify the structure

🔑 Key concepts

  • Binary decisions use if / else — when there are exactly two mutually exclusive outcomes, a single condition splits the flow in two (discount vs. none, even vs. odd).
  • Classification uses a chain — when there are more than two outcomes, if / else if / … / else tests options in order and stops at the first match.
  • Always include a catch-all else — the final branch (Category: Unknown.) handles every input you did not explicitly list, so nothing falls through silently.
  • Pick the structure from the question, not by habit — "How many distinct outcomes are there?" decides binary vs. multi-way before you write a line of code.
  • Modulo (%) is the even/odd test — n % 2 == 0 is the idiomatic, language-independent way to check divisibility by 2.
